2 Configuration

There are three kinds of configuration data in Whatnow:

  1. Defaults for various program options.

  2. Data the is required for a local installation of Whatnow, such as the specifics of where to find the various servers. This kind of data is held in Racket modules. See Local configuration.

  3. Data that should not be made public, such as authentication keys. These settings are stored in configuration files that must be created by the user and stored in a directory following the XDG standard. See Secrets.

2.2 Secrets

 (require whatnow/config)
The whatnow/config library retrieves local configuration information from a config file. This information is mainly account authentication details which should not be stored with the source code.

struct

(struct server-account (id token)
    #:transparent)
  id : string?
  token : string?
A structure type for internet server accounts. A server account consists of an account identifier and an authentication token. The details of "id" and "token" are specific to the server.

procedure

(config-get-accounts [secret-file-name]) → 
(listof
pair?)
  secret-file-name : string? = "secrets.txt"
Retrieve secrets from a configuration file. The configuration file is looked for in the directories specified by the (part ("(lib basedir/basedir.scrbl)" "XDG_Basedir_Specification")) using list-config-files.

Calling config-get-accounts returns an association list. Presently there is a single key, 'forecast, and the value is a server account structure.
